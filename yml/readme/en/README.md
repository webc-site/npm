## Usage & Test

A minimalist, high-performance, single-pass YAML parser based on a state machine.

### 🚀 LLM Fault Tolerance Design

This parser is specifically engineered to handle **YAML generated by Large Language Models (LLMs)**. Regular YAML parsers often crash when processing LLM-generated texts due to minor formatting errors.

Our parser features robust fault tolerance, gracefully resolving the following common issues:

1. **Irregular Indentation (Bad Indent)**: Even with irregular spaces, the state machine adaptively reconstructs the tree hierarchy based on the relative changes in indent depth.
2. **Unclosed Quotes (Bad Quote)**: If a quote is not closed (e.g. `key: "hello`), it will be safely treated as a raw string instead of raising a syntax error.
3. **Empty Values or Comments only**: When a key is declared without a value or followed only by inline comments (e.g. `key:  # inline comment`), the parser automatically initiates it as an empty nested object.
4. **EOF on Multiline Blocks**: Graciously handles cases where the multiline block indicator `|` is at the end of file without further contents.
5. **Robust Input Filtering**: Safely handles irregular empty values (`null`/`undefined`/`""`) and various comments.

### Installation

```bash
bun i @1-/yml
```

### Usage

```javascript
import load from "@1-/yml/load.js";
import loads from "@1-/yml/loads.js";

// Parse YAML string
const obj = loads("a: 1");

// Parse YAML file
const fileObj = load("./conf.yml");
```
